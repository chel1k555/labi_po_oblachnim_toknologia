# Отчет по облачной лабораторной № 1

Был проведен анализ биллинга от провайдера с данными AWS из файла team 5, потому что это мое любимое число. На основе документации от провайдера и учитывая данные из Product Code и Usage Type, были причесаны) данные для каждого Амазон сервиса в такой вид -> |IT Tower | Service Family | Service Type | Service Sub Type | Service Usage Type|.

В итоге данные можно было отнести к какому-то из классов: DataBase, Storage, Compute, CLoud Services или Support.


**Amazon AppStream**

Это полностью управляемый сервис потоковой передачи приложений и рабочих столов (Desktop-as-a-Service). он позволяет пользователям получать доступ к установленным на сервере программам через обычный веб-браузер.


**Amazon DocumentDB**

Это быстрая, масштабируемая и полностью управляемая база данных документов, совместимая с MongoDB. Она использует распределенную отказоустойчивую систему хранения данных.


**AWS IoT Core**

Управляемая облачная платформа, которая позволяет подключать миллиарды IoT-устройств (датчиков, актуаторов) и безопасно обмениваться данными с облачными приложениями.


**Amazon Machine Learning**

Сервис, который упрощает создание прогностических моделей для разработчиков любого уровня. Он предоставляет визуальные инструменты и мастера для анализа данных и обучения моделей без необходимости глубоко знать математику нейросетей.


**Amazon EC2**

Фундаментальный сервис, предоставляющий виртуальные серверы (инстансы) в облаке. Это уровень IaaS (инфраструктура как сервис), где вы получаете полный контроль над операционной системой.


**Amazon Personalize**

Специализированный сервис машинного обучения для создания систем персонализированных рекомендаций в реальном времени. Он использует те же технологии, что и Amazon.com.

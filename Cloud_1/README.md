# Отчет по облачной лабораторной № 1

Был проведен анализ биллинга от провайдера с данными AWS из файла team 5, потому что это мое любимое число. На основе документации от провайдера и учитывая данные из Product Code и Usage Type, были причесаны) данные для каждого Амазон сервиса в такой вид -> |IT Tower | Service Family | Service Type | Service Sub Type | Service Usage Type|.

В итоге данные можно было отнести к какому-то из классов: DataBase, Storage, Compute, CLoud Services или Support.

____________________________________________________________________________________________________________________________________________________

**Amazon AppStream**

Облачный сервис для потоковой доставки приложений пользователям.
Позволяет запускать настольные приложения на серверах AWS и передавать изображение и ввод пользователю через браузер без установки ПО. Используется для удалённого доступа к корпоративным приложениям и виртуальным рабочим средам.


**Amazon DocumentDB**

Управляемая документо-ориентированная база данных, совместимая с API MongoDB. Предназначена для хранения и обработки JSON-документов с автоматическим масштабированием, резервным копированием и высокой доступностью.


**AWS IoT Core**

Набор сервисов для подключения, управления и обмена данными с IoT-устройствами. Обеспечивает безопасную аутентификацию устройств, приём телеметрии, обработку событий и интеграцию с другими сервисами AWS.


**Amazon Machine Learning**

Сервис для построения, обучения и применения моделей машинного обучения. Позволяет анализировать данные и получать прогнозы без глубоких знаний ML, предоставляя готовые алгоритмы классификации и регрессии.


**Amazon EC2**

Сервис предоставления виртуальных серверов в облаке. Позволяет запускать вычислительные ресурсы с гибкой настройкой процессоров, памяти, хранилища и сетевых параметров, масштабировать нагрузку и оплачивать фактическое время использования.


**Amazon Personalize**

Сервис машинного обучения для персонализации пользовательского опыта. Используется для создания рекомендаций товаров, контента или действий на основе поведения пользователей в реальном времени, без необходимости самостоятельно разрабатывать ML-модели.
